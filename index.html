Group Processes

Based on assignment 2 many things could be improved upon. The main area of failure we experienced was with a lack of communication which resulted in members taking on more work than others and little to no work being done by others. This isn’t just a communication problem but also a commitment and dedication issue. The first way we plan on tackling this and improving this is by having more meetings a week with a set time that everyone in the group is aware of.
We also discussed that everyone should be checking the team's chat at least once a day. This would help to ensure that everyone is up to date with the latest information. On top of this a new feature in Microsoft Teams was introduced with a plugin that allows for members to assign roles for them to complete and mark off once they complete them. This allows for everyone to see what is being worked on, hasn’t been done and what has been completed.

Overall, the team has decided that in order to complete this assignment successfully a major improvement on group communication and collaboration will need to be made.

Tools

In this assignment our group has been able to successfully produce 3 artifacts 2 of which being a prototype. We have included a flow diagram to illustrate proper navigation of the app and the different paths a user can take when using our app. It also is there to help illustrate how each component of the app will work together using various techniques like asking for user input, radio buttons, drop down menus and buttons in general. As well as this we have provided a Figma prototype which acts to show the user interface of the app and further illustrate the navigation. This has also been used to help with the 2nd prototype that was developed with Flutter. Our flutter prototype doesn’t focus too much on the user interface but reflects some functionality that the app when fully produced will contain. The functionality that we wished to flesh out in this prototype are the random task being selected, and the various interactive elements such as buttons and check lists.
On top of this visual studio code was used to edit our previous website which was made in Assignment 2. To save time we simply just replaced the information with that we have, as opposed to creating a new website from scratch.
